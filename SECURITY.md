# 安全政策

## 报告安全问题

如果你发现了安全问题，请不要直接公开披露真实密钥、服务端地址的敏感参数、用户数据或可复现的攻击细节。请先通过私下渠道联系维护者，或在 issue 中提交不包含敏感信息的概述。

报告时建议包含：

- 受影响的版本或提交。
- 问题类型和影响范围。
- 最小复现步骤。
- 已确认不会泄露真实 API Key 或用户数据的日志片段。

## 本地凭据

应用会在本机 Tauri 应用数据目录保存 API Key 和用户设置。请不要把自己的 `connection.json`、`settings.json`、截图中的 Key、打包签名文件或本地生成结果提交到公开仓库。
